Всем привет,

Я разработал плагин для Obsidian, который решает именно эту проблему: **Linux Image Rendering Fix**.

## Корень проблемы

Как уже упоминалось в этой ветке, это upstream-баг Electron/Chromium. Протокол `app://` не может разрешить пути, содержащие не-ASCII символы (китайские, русские, эмодзи и т.д.). Плюс некоторые сборки Electron для Linux (особенно Snap и Flatpak) не имеют кодеков изображений (PNG, JPEG, WebP). Этой проблеме уже три года с версии Obsidian v1.0.0, и на уровне приложения она до сих пор не исправлена.

## Как плагин это чинит

Плагин перехватывает рендеринг Markdown через `MarkdownPostProcessor`, находит каждый элемент `<img>`, читает бинарные данные файла изображения напрямую с диска через `vault.adapter.readBinary()`, затем заменяет `src` на Blob URL (`blob:...`). Это **полностью обходит сломанный протокол `app://`** и не зависит от кодеков Electron.

## Возможности

- **Автоматическая обработка** — изображения подменяются при открытии страницы, переключении вкладок и изменении файла
- **Поддерживаемые форматы:** PNG, JPEG, GIF, WebP, SVG, BMP, ICO
- **Команда Reprocess Images** — ручное обновление текущего представления
- **Безопасность:** работает только внутри vault, ноль сетевых запросов
- **Нулевая настройка:** установил и работает

## Покрытие

Я протестировал все сценарии, упомянутые в этой ветке и за её пределами:

- Картинка, вставленная в заметку — работает
- Превью при наведении (popover) — работает
- Прямой просмотр файла изображения — работает
- Изображения на неактивных/фоновых вкладках — работает (ловит MutationObserver)

Возможно есть крайние случаи, которые я не нашёл, но пока я не видел битого изображения, которое плагин не подхватил.

## Известные нюансы

Некоторые сценарии срабатывают с небольшой задержкой при первой загрузке — системе нужен момент на «разогрев», прежде чем плагин включится в работу.

## Установка

Settings → Community Plugins → Browse → поиск **Linux Image Rendering Fix** → Install → Enable

**Репозитории:** [GitHub](https://github.com/Evgene-Kopylov/linux-image-rendering-fix) | [GitLab](https://gitlab.com/Evgene-Kopylov/linux-image-rendering-fix)

**Лицензия:** LGPL-3.0, полностью бесплатно и открыто.

Проверено на Ubuntu 24.04 (DEB-пакет). Другие дистрибутивы (Pop!_OS, Linux Mint, Fedora, Arch и т.д.) и форматы пакетов (Snap, Flatpak, AppImage) тоже поддерживаются — потому что корень проблемы в upstream-баге Electron, а не в конкретном дистрибутиве.

## Преимущества перед другими обходными путями

- **Не нужно** переименовывать папки на английский
- **Не нужно** прописывать `LANG`/`LC_ALL` в переменные окружения
- **Не нужно** менять способ установки (Snap → DEB)
- **Один плагин — все проблемы решены**
